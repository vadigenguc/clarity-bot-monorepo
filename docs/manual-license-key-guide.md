# Manual License Key Guide

This guide outlines the process for manually adding a license key from Polar to the Supabase database after a user completes a purchase. This is a temporary workaround until the webhook automation is fully resolved.

### Process Overview

The goal is to link a license key provided by Polar to a user who has paid. This involves two main steps: identifying the user in the `founding_members` table and then creating a corresponding entry in the `license_keys` table.

---

### Step 1: Identify the User in `founding_members`

1.  After a user completes a purchase, you will find their payment details, including their **email address**, on the Polar dashboard.
2.  Go to your Supabase project and navigate to the **Table Editor** section.
3.  Select the `founding_members` table from the list.
4.  Use the filter or search functionality to find the row corresponding to the user's email address.
5.  Once you have located the correct user, copy the `id` value from that row. This UUID is the unique identifier for the founding member and will be used as the `client_reference_id` to link the tables.

### Step 2: Add the License Key to `license_keys`

1.  While still in the Supabase Table Editor, select the `license_keys` table from the list.
2.  Click the **"+ Insert row"** button to open the form for adding a new record.
3.  Fill in the following columns for the new row:
    *   **`license_key`**: Paste the license key that was generated by Polar.
    *   **`customer_email`**: Enter the user's email address again for reference.
    *   **`tier`**: Set this value to `Founder`.
    *   **`client_reference_id`**: **This is the most critical step.** Paste the `id` you copied from the `founding_members` table in Step 1. This creates the link between the user and their key.
    *   The other columns (`id`, `is_redeemed`, `created_at`, etc.) can be left to their default values; Supabase will handle them automatically.
4.  Click **"Save"** to create the new license key record.

The user is now correctly set up in the database with their license key.
