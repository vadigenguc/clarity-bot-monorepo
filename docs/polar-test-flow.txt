Based on the actions our Netlify function (`polar-webhook.js`) performs, it only needs to __read__ data from Polar. It receives a webhook and then uses the Polar SDK to verify the webhook's signature. It doesn't write or change any data within your Polar account.

Therefore, you should select the most restrictive, read-only scopes available. Look for scopes related to:

- `webhooks:read`
- `subscriptions:read`
- `customers:read`
- `orders:read`

You should __not__ need any `write` or `admin` permissions. Always grant the minimum permissions necessary for an application to function. In our case, that is just reading the webhook data.
